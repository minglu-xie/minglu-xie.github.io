Configure jupyter notebook settings in Linux server to achieve remote access

2026-02-18 3 min read

I am doing my master thesis in SciLife lab, Uppsala University. We have our own Linux Server in the lab. Usually you don't have the root access which will make it difficult to configure Bioinformatic analysis environment sometimes.

Recently, my colleague asked me to configure my own Jupyter Notebook setting and install R and Python to achieve remote access from local. Here I installed R 4.2.0 and Python 3.11.0 for my Jupyter Notebook.

Notice: I mainly use mamba instead of conda now. You can find how to use mamba here: https://mamba.readthedocs.io/en/latest/user_guide/mamba.html

Prerequisites:

  • Basic knowledge of Linux
  • Basic knowledge of conda/mamba

Setup

  1. Create and activate a virtual environment (R420):
conda create --name r420
conda activate r420
mamba install -c conda-forge r-base=4.2.0
mamba install python=3.11.0 -c conda-forge
  1. Install Jupyter Notebook:
pip install notebook

To uninstall:

pip uninstall notebook
  1. Generate the Jupyter configuration file:
jupyter notebook --generate-config
  1. Set the password:
jupyter notebook password
  1. Edit the configuration file:
vim ~/.jupyter/jupyter_notebook_config.py

Add or modify the following lines:

c.NotebookApp.ip = '*'
c.NotebookApp.open_browser = False
c.NotebookApp.port = 8998
c.NotebookApp.allow_remote_access = True
  1. Start Jupyter Notebook:
jupyter notebook

You can now access it from your local browser using the server IP address and the port you set.

If you want it to keep running in the background:

nohup jupyter notebook &
  1. Check available kernels:
jupyter kernelspec list

Install R Kernel in Jupyter Notebook

Normally you can install IRkernel inside the R console using install.packages(), but since I do not have root access, dependency errors occurred. Therefore, using mamba is recommended:

mamba search r-irkernel
mamba install -c conda-forge r-irkernel=1.3.1

After installation, open R and run:

IRkernel::installspec(name = "R", displayname = "R_4.2.0")

Check the Jupyter kernels again — you should now see both Python and R kernels available.

Thank you for reading.
